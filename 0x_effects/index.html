The primary reason this blog post exists is to help me understand exactly what is happening under the hood. We're going to build up the shader bit by bit. Step X: Texture Flip

WebGL has this weird behaviour where textures are typically flipped. We can actually solve this by getting the rendered from p5 and calling a function that tells WebGL if we want textures fippped.

But since we have power of shaders, we can do the flip within the shader itself. We're going to remap the y-coordinates from 0..1 to 1..0.

Step X: Repeating the Texture

Examine the final demo and you can see that the texture is repeated. We can easily achieve this by using the built-in mod function. Not surprisingly, mod accepts 2 values.

we scale the x component by 4 then modulus it, thus ensuring the value ranges for 0 to 1.
